2.读取原始单细胞测序数据

以10X为例，我们测序得到的数据一般是包含三个文件，基因表达矩阵matrix.mtx, 细胞注释信息barcodes.tsv, 和基因注释信息genes.tsv。

tsv扩展名为制表符间隔的文件，它们储存空间相对来说比较小，所以读取的时候需要的时间较小，不考虑处理它。

而mtx扩展名文件其实是The Matrix Market File Format 类型的文件， 扩展名不是十分的重要，MM文件是固定的文件结构，它是一种类似COO稀疏矩阵的存储数据的方式，并利用%%MatrixMarket object format field symmetry进行标识。它的格式可以查看这个网址的说明。这个数据是很大的十几Gb甚至几百Gb以上，所以我们要有合适的读取这个数据的办法。

我们做分析的时候主要使用的是R和Python。我们分析的数据的结果必须做到，R到Python的数据可以相互的，快速的读取， 根据主流的单细胞分析软件，我们暂时先用Seurat for R， Scanpy for Python

  • for R

3. 稀疏矩阵的写出写入

为了达到R和Python数据的可以，快速地，友好地相互读取，我们必须利用一些两者都支持地文件格式，并且读取写入都要快。

我们暂时先利用hdf5文件，它是将数据转换成二进制地数据进行存储，这样不管读取还是写入都有很快地提升，而且两种语言都是兼容地。

4. 定制hdf5文件的目录格式

在数据分析中，我们主要需要保存的信息有：

保存数据的后缀名，拟用.hdf5

  1. raw data， 原始稀疏矩阵
  2. normalized data，标准化后稀疏矩阵
  3. scale data，scale后的稠密矩阵，一般选择特定的高变异基因
  4. cell annotation dataframe，细胞注释信息矩阵
  5. gene annotation dataframe， 基因注释信息矩阵
  6. 不同群的差异基因
  7. 其他注释信息，如pca，umap矩阵，这个格式和4，5一样

hdf5目录格式：

  1. RData：(group)

    • Value (dataset)

    • Indices (dataset)

    • Indptr (dataset)

    • Dim_CSC (dataset)

    • Dimnames (group)

      • Obs
      • Var

  2. NData: (group)

    • Value (dataset)

    • Indices (dataset)

    • Indptr (dataset)

    • Dim_CSC (dataset)

    • Dimnames (group)

      • Obs
      • Var

  3. SData: (group)

    • Matrix (dataset)

    • Dim_CSC (dataset)

    • Dimnames (group)

      • Obs
      • Var

  4. Annotate: (group)

    • Cell_Annotate: (dataset)

      • rownames (attributes)
      • colnames (attributes)

  5.
